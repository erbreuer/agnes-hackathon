# F05 — Room Renderer (render_room)

> INITIAL spec. Read together with CLAUDE.md (project memory + contract).

## Goal
A render agent that composites the uploaded room photo together with the selected
real product images into three distinct redesigned renders by calling the Agnes
image generation API (agnes-image-2.0-flash).

## Contract (must match CLAUDE.md exactly)

      async def render_room(
          room_b64: str,
          products: list[dict],
          design_summary: str
      ) -> list[str]

- `room_b64` is the raw base64 string of the room photo (no "data:" prefix).
- `products` is the JSON list returned by scout_products. Each item is a dict
  with at minimum these keys (matching the /design API contract in CLAUDE.md):
      { "name": str, "price": float, "link": str, "image": str }
  `image` is a SerpApi product thumbnail URL — pass it through untouched.
- `design_summary` is the text paragraph returned by plan_design.
- Returns a list of exactly 3 render URLs (strings). On partial failure, pad with
  any successful URLs and log warnings; never crash the caller.

## Requirements

1. Build a base64 data URI from room_b64:
       room_uri = "data:image/jpeg;base64," + room_b64

2. Extract product image URLs: `[p["image"] for p in products if p.get("image")]`,
   cap at 5 (Agnes may reject very long image arrays).

3. Build the image array for extra_body.image:
       image_array = [room_uri] + product_image_urls

4. Build the render prompt from design_summary:
   - Focus the model on placing the selected pieces into the existing room.
   - Preserve the room's layout, windows, doors, and perspective — do NOT invent
     new architecture.
   - Frame the result as a styled "shop this look" visualisation.
   - Keep the prompt under 300 characters so flash models stay on task.
   Suggested template:
       "Interior design render. Place these furniture pieces naturally into the
        existing room, preserving its layout, windows and perspective.
        Style: {design_summary[:180]}. Photorealistic, warm lighting."

5. Call agnes.image() THREE times with that same prompt and image_array.
   Collect the returned URL list from each call (each call returns a list; take
   index 0 of each). Accumulate into a renders list.

6. Return renders (length 3 if all calls succeeded; shorter if some failed — the
   caller handles display gracefully).

7. Timeout behaviour is already handled inside agnes.image() (60s). Do NOT add a
   separate timeout here.

## Agnes image call (reference — agnes-image-2.0-flash)

      POST https://apihub.agnes-ai.com/v1/images/generations
      Authorization: Bearer $AGNES_API_KEY
      {
        "model": "agnes-image-2.0-flash",
        "prompt": "<render instruction>",
        "size": "1024x768",
        "extra_body": {
          "image": [
            "data:image/jpeg;base64,<room_b64>",
            "https://product1-from-serpapi.jpg",
            "https://product2-from-serpapi.jpg"
          ],
          "response_format": "url"
        }
      }

GOTCHA: response_format MUST be inside extra_body, never at the top level.
GOTCHA: the room goes first in the image array so the model treats it as the
        base scene; product images follow as reference.

## Gotchas
- Keep product_image_urls capped at 5 — longer arrays can hit Agnes payload limits.
- Call image() three times sequentially (not concurrently) to avoid rate-limit
  errors on the free tier.
- Each image() call returns a LIST; take [0] if non-empty, otherwise skip and log.
- design_summary may be long — slice to 180 chars in the prompt template so the
  total prompt stays under 300 chars.
- Product images come directly from SerpApi URLs — pass them through untouched,
  never re-host or re-encode.

## Validation

1. Base64-encodes scripts/sample_room.jpg.
2. Uses a hard-coded stub products list (2-3 items with real-looking image URLs).
3. Uses a hard-coded design_summary string.
4. Calls render_room and pretty-prints the result.

      python scripts/test_agents.py render

Expected: a list of 3 https:// URLs that point to valid Agnes-generated images.
Run it once — if all three URLs load in a browser the feature is done.
