## Contract (must match CLAUDE.md exactly)

      async def image(prompt: str,
                      image_urls: list[str] = [],
                      size: str = "1024x768",
                      model: str = "agnes-image-2.0-flash") -> list[str]

      async def video(prompt: str,
                      image_url: str | None = None,
                      model: str = "agnes-video-v2.0") -> str | None

- image() returns a list of render URLs (usually 1 per call).
- video() returns a single finished video_url, or None on failure/timeout.

## Requirements

### image()
1. POST to {BASE}/images/generations with the Bearer header.
2. Body shape (follow Agnes docs exactly):
   - top level: model, prompt, size
   - extra_body: { image: image_urls, response_format: "url" }
3. image_urls may mix base64 DATA URIs (the room photo) and plain https URLs
   (product images) — pass them through untouched.
4. Parse the response defensively: read each item's url under response["data"].
   If an item only has b64_json, return it as a data URI instead. Return the list.
5. Timeout: 60s.

### video() (asynchronous — create then poll)
1. POST to {BASE}/videos with model, prompt, and (if given) image, num_frames=121,
   frame_rate=24. Read the returned task id.
2. Poll GET {BASE}/videos/{id} every 5s until the task is done; return video_url.
3. Stop polling after ~5 minutes (e.g. 60 tries); return None on timeout or failure.
4. Timeout per request: 30s.

### Both
- Reuse the same auth header + httpx pattern from F00's chat().
- Key from environment (already loaded). Never hardcode or log key values.
- On error, return an empty list (image) or None (video) — never crash the caller.

## Agnes image call (reference — agnes-image-2.0-flash)

      POST https://apihub.agnes-ai.com/v1/images/generations
      Authorization: Bearer $AGNES_API_KEY
      {
        "model": "agnes-image-2.0-flash",
        "prompt": "<instruction>",
        "size": "1024x768",
        "extra_body": {
          "image": ["data:image/jpeg;base64,<room>", "https://product1.jpg"],
          "response_format": "url"
        }
      }

GOTCHA: response_format goes INSIDE extra_body, never at the top level.
Expected response shape: { "data": [ { "url": "https://..." } ] }

## Agnes video call (reference — agnes-video-v2.0, async)

      # 1) create the task
      POST https://apihub.agnes-ai.com/v1/videos
      {
        "model": "agnes-video-v2.0",
        "prompt": "<instruction>",
        "image": "https://render.jpg",   # optional (image-to-video)
        "num_frames": 121,
        "frame_rate": 24
      }
      # -> returns a task id

      # 2) poll until done
      GET https://apihub.agnes-ai.com/v1/videos/{id}
      # -> { "status": "...", "video_url": "..." }   (read video_url when completed)

## Gotchas
- response_format MUST be inside extra_body for the image endpoint.
- The image array mixes data URIs and plain URLs — don't transform either.
- Video is async: a POST that returns a URL directly is wrong; you must poll.
- Field names in the video response (id vs task_id, status values, video_url) can
  vary — LOG the raw JSON on the first create + first poll, confirm the names, then
  parse. Don't guess silently.
- Use a real httpx timeout object; the default can hang the whole request.
