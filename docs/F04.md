# F04 — Product Scout (`scout_products`)

**Goal:** Retrieve real, purchasable products within budget via SerpApi, with a local
catalog fallback so the demo never breaks.

**Depends on:** F03 (plan_design liefert die Shopping-Liste mit per-item Budget-Caps)

---

## Kontext

`scout_products` ist Schritt 3 der sequenziellen Pipeline:

```
analyze_space → plan_design → scout_products → render_room
                                    ↑
                              dieser Feature
```

Für jedes Item aus dem `plan_design`-Output werden echte, kaufbare Produkte gesucht.
Erst SerpApi (Google Shopping), bei Fehler/Ratelimit/leerem Ergebnis: Fallback auf `catalog.json`.
Das Ergebnis geht direkt als `extra_body.image`-URLs in den Agnes Image-Render.

## Requirements

### R1 — SerpApi-Suche
```
GET https://serpapi.com/search
  ?engine=google_shopping
  &q=<search_query>
  &max_price=<int>
  &api_key=$SERPAPI_KEY
```

- Pro Item: top 1–2 `shopping_results` nehmen
- Felder mappen auf: `{name, price, link, image}`
  - `title` → `name`
  - `extracted_price` → `price`
  - `product_link` → `link`
  - `thumbnail` → `image`

### R2 — Fallback auf `catalog.json`
Bei **jedem** Fehler (Exception, leere Ergebnisse, HTTP-Fehler, Ratelimit):
- `catalog.json` einlesen
- Nach `category` + `price ≤ budget_cap` filtern
- 1–2 Treffer zurückgeben

> Die Pipeline muss **immer** Produkte zurückgeben — auch mit falschem API-Key.

### R3 — `catalog.json` Seed-Daten
16–20 Produkte über folgende Kategorien verteilt:

| Kategorie | Mindestanzahl |
|---|---|
| sofa | 2 |
| chair | 3 |
| table | 2 |
| rug | 2 |
| lamp | 3 |
| shelf | 2 |
| art | 2 |
| plant | 2 |

Jedes Produkt hat: `name`, `category`, `price` (int), `link`, `image` (echte Bild-URL).

---

## Output-Shape

```json
[
  {
    "name": "Linen Accent Chair",
    "price": 299,
    "link": "https://...",
    "image": "https://..."
  }
]
```

Eine Liste — ein Eintrag pro gesuchtem Item. `image` ist eine direkte URL
(wird später als `extra_body.image` in `render_room` übergeben — **nie re-hosten**).

---

## Implementation

```python
async def scout_products(items: list[dict]) -> list[dict]:
    """items = [{"search_query": str, "budget_cap": int, "category": str}, ...]"""
    results = []
    async with httpx.AsyncClient(timeout=15) as client:
        for item in items:
            product = await _fetch_serpapi(client, item) or _fallback(item)
            if product:
                results.append(product)
    return results
```

- `_fetch_serpapi` → einzelner SerpApi-Call, gibt `dict | None` zurück
- `_fallback` → liest `catalog.json`, filtert, gibt bestes Match zurück
- Beide Hilfsfunktionen unter ~20 Zeilen halten

---

## Konventionen (aus CLAUDE.md)

- `httpx.AsyncClient`, Timeout 15 s für externe Suche
- Gesamte SerpApi-Logik in `try/except` — niemals den Request crashen
- Key kommt aus `.env` via `python-dotenv` — nie hardcoden oder loggen
- Funktion unter ~40 Zeilen; Hilfsfunktionen auslagern
