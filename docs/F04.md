F04 — Product Scout (scout_products)
Goal: Retrieve real, purchasable products within budget via SerpApi, with a local catalog fallback so the demo never breaks.
Depends on: F03.
Files to create / modify
•	backend/agents.py
•	backend/catalog.json
•	backend/scripts/test_agents.py
Requirements
17.	async scout_products(items): for each item, query SerpApi Google Shopping with q=search_query and max_price; take the top 1–2 results mapped to {name, price, link, image}.
18.	On any error, empty result, or rate-limit: read catalog.json and filter by category + price. The pipeline must always return products.
19.	Seed catalog.json with ~16–20 sample products across categories (sofa, chair, table, rug, lamp, shelf, art, plant) — each with name, category, price, link, image (a real image URL).
Implementation notes
GET https://serpapi.com/search
  ?engine=google_shopping
  &q=<search_query>
  &max_price=<int>
  &api_key=$SERPAPI_KEY
# use: shopping_results[].title / .extracted_price / .product_link / .thumbnail
Wrap the whole SerpApi call in try/except and fall through to catalog.json. Test with a fake key to confirm the fallback works.
