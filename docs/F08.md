# F08 — Frontend (`App.tsx` + `api.js`)

**Goal:** A beautiful single-page UI for the upload → render → refine loop.
Swedish-inspired aesthetic: calm, airy, warm — clean blues, warm whites, natural wood tones.

**Depends on:** F06 (`/design`), F07 (`/refine`)

---

| Token | Value | Usage |
|---|---|---|
| `--blue-deep` | `#1B3A52` | Headings, primary CTA background |

### Typography
- Font: `'Inter', system-ui, sans-serif` — loaded via Google Fonts or already in Vite
- Headings: `font-weight: 300` (light), letter-spacing `0.04em` — very Scandinavian
- Body: `font-size: 15px`, `line-height: 1.6`

### Layout feel
- Max width `860px`, centered, plenty of whitespace
- Generous padding (`2rem`–`3rem` sections)
- Rounded corners `12px` on cards, `8px` on inputs/buttons
- Subtle `box-shadow: 0 2px 12px rgba(27,58,82,0.07)` on cards

## Requirements

### R1 — Upload Form
A single clean form card with:
- **Room photo upload** — drag-and-drop zone or click-to-browse. Read file to base64. Show a small preview thumbnail once chosen.
- **Prompt** — textarea, placeholder: `"e.g. make it cozy and Scandinavian"`
- **Budget** — number input with `$` prefix, placeholder `1500`
- **Optional ref image URLs** — a small collapsible "Advanced" section, comma-separated text input
- **Generate button** — full-width, `--blue-deep` background, white text. Shows a spinner + "Generating…" while loading.

### R2 — Results: Render Gallery
Three render images displayed in a responsive grid (3 columns on desktop, 1 on mobile).
Each image:
- `border-radius: 12px`, subtle shadow
- Fills its column with `object-fit: cover`
- Clicking opens the full image in a new tab

Above the grid: a short `design_summary` text in italic, muted style.

### R3 — Product Cards
A horizontal scrollable row (or 2-column grid on desktop) of product cards below the renders.
Each card shows:
- Product image (`80×80px`, `border-radius: 8px`, `object-fit: cover`)
- Name (truncated at 2 lines)
- Price in `$XX`
- **"Buy →"** link — opens `product.link` in a new tab, styled as a small pill button in `--blue-mid`

### R4 — Refine Bar
A sticky-ish section below the results with:
- Text input: `"Refine the design… (e.g. cheaper sofa, add more plants)"`
- **Refine button** — same style as Generate, but outlined/secondary style
- On submit: calls `POST /refine` with `session_id` + `feedback`, replaces renders + products

### R5 — States
| State | UI |
|---|---|
| Idle | Only the form visible |
| Loading | Spinner overlay on the button, form inputs disabled |
| Error | A small red banner below the form: `"Something went wrong — try again"` |
| Results | Form collapses/scrolls up, results + refine bar appear |

## Implementation notes

- One screen, no router, no state library — `useState` + `useCallback` only
- All styles in `App.css` using CSS custom properties (variables defined on `:root`)
- No new npm dependencies — plain React + fetch
- File → base64: use `FileReader.readAsDataURL`, strip the `data:...;base64,` prefix before sending
- Disable the Generate button while `loading === true`
