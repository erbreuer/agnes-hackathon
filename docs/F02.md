# F02 — Space Analyst (analyze_space)

## Goal

that the rest of the pipeline can plan against.

## Contract (must match CLAUDE.md exactly)

      async def analyze_space(room_b64: str) -> dict

`room_b64` is the raw base64 string of the uploaded room photo (no "data:" prefix).
Returns a dict with exactly these keys:

      {
        "style":          str,     # e.g. "mid-century modern"
        "palette":        [str],   # 3-5 dominant colors (words or hex)
        "lighting":       str,     # e.g. "bright, large north-facing window"
        "fixed_elements": [str],   # can't move: windows, doors, radiator
        "approx_size":    str,     # rough scale, e.g. "small ~10 sqm"
        "keep":           [str],   # existing items worth keeping
        "replace":        [str]    # existing items the user likely replaces
      }

## Requirements
1. Call the Agnes chat model with vision: send a content ARRAY containing a text
   instruction plus an image_url whose url is a base64 DATA URI built from room_b64.
2. Instruct the model to return ONLY valid JSON matching the schema above — no prose,
   no markdown, no code fences.
3. Parse defensively: strip any ``` or ```json fences and surrounding whitespace
   before json.loads. If parsing fails, retry once with a stricter "JSON only"
   instruction; if it still fails, return a minimal valid dict (empty lists,
   "unknown" strings). Never crash the request.
4. Reuse chat() from agnes.py for the call — do NOT re-implement the HTTP request.
5. Key comes from the environment (handled in agnes.py). Never hardcode or log keys.

## Agnes vision call (reference — agnes-2.0-flash)
Build the data URI as:  "data:image/jpeg;base64," + room_b64
Then the messages payload that chat() sends to POST /chat/completions:

      messages = [{
        "role": "user",
        "content": [
          {"type": "text", "text": "<strict JSON instruction>"},
          {"type": "image_url",
           "image_url": {"url": "data:image/jpeg;base64,<room_b64>"}}
        ]
      }]

Suggested instruction text:

      "You are an interior design analyst. Look at this room photo and return a JSON
       object describing it, using exactly these keys: style, palette, lighting,
       fixed_elements, approx_size, keep, replace. The palette, fixed_elements, keep,
       and replace values are arrays of short strings. Respond with raw JSON only —
       no markdown, no code fences, no explanation."

## Gotchas
- The image goes in a content ARRAY (text + image_url), never a plain string.
- Flash models often wrap JSON in ```json fences — always strip before parsing.
- Keep the instruction short and explicit; long prompts make flash models drift.
- approx_size is a rough estimate from the photo; don't try to be precise.
