# F06 — Orchestrator + POST /design

> INITIAL spec. Read together with CLAUDE.md (project memory + contract).

## Goal
Wire the four agents (F02-F05) into a single POST /design endpoint with in-memory
sessions, so one request runs the whole pipeline and returns renders + products.

## Owner / branch
Owner: A    ·    Branch: feat/F06-orchestrator

## Depends on
F02-F05 (analyze_space, plan_design, scout_products, render_room). Build against the
F00 STUBS first — /design should run end-to-end with stub data immediately, and the
real agents slot in automatically as their PRs merge into dev.

## Files
- Modify:  backend/main.py   (add the /design endpoint, request model, session store)

## Contract (must match CLAUDE.md exactly)

Request — POST /design:

      {
        "room_image": str,      # raw base64 of the room photo (no "data:" prefix)
        "ref_images": [str],    # optional style reference URLs (may be [])
        "prompt":     str,      # e.g. "cosy mid-century living room"
        "budget":     number    # total budget
      }

Response:

      {
        "session_id":  str,
        "space_brief": { ... },                 # from analyze_space
        "renders":     [url, url, url],          # from render_room
        "products":    [ {name, price, link, image}, ... ]   # from scout_products
      }

## Requirements
1. Define a Pydantic request model DesignRequest with the four fields above
   (ref_images defaults to []).
2. In the async /design handler, run the pipeline in order, awaiting each agent:
     space_brief = await analyze_space(req.room_image)
     plan        = await plan_design(space_brief, req.prompt, req.budget)
     products    = await scout_products(plan["items"])
     renders     = await render_room(req.room_image, products, plan["design_summary"])
   (plan_design returns {design_summary, items}.)
3. Create session_id = uuid4().hex. Store EVERYTHING F07 will need to refine later:
     SESSIONS[session_id] = {
       "room_b64":    req.room_image,
       "space_brief": space_brief,
       "prompt":      req.prompt,
       "budget":      req.budget,
       "products":    products,
     }
   SESSIONS is a plain module-level dict. No database.
4. Return {session_id, space_brief, renders, products}.
5. Import the agents from the agents package (from agents import analyze_space,
   plan_design, scout_products, render_room).
6. Reuse the FastAPI app + CORS already set up in F00. Don't re-create them.
7. On any agent failure, return a clear JSON error (HTTP 500 with {"error": "..."}),
   never an unhandled crash.

## Implementation notes
- Storing the full room_b64 in memory per session is fine for the hackathon — it's
  what lets /refine re-render the same room without another upload.
- ref_images: accept it for forward-compat. Wiring it into rendering is optional for
  the MVP and must be coordinated with D (render_room's signature) — you may leave it
  unused for now. Do NOT change agent signatures here.
- Keep the handler thin: it orchestrates, it doesn't contain agent logic.

## Validation
With uvicorn running and a sample image at backend/scripts/sample_room.jpg:

      cd backend
      python - << 'PY'
      import base64, json
      b = base64.b64encode(open("scripts/sample_room.jpg","rb").read()).decode()
      json.dump({"room_image": b, "ref_images": [],
                 "prompt": "cosy mid-century living room", "budget": 1500},
                open("body.json","w"))
      print("wrote body.json")
      PY

      curl -X POST http://localhost:8000/design \
        -H "Content-Type: application/json" -d @body.json

Expected: JSON with a session_id, a space_brief, 3 renders, and a products list.
(Works with stub agents too — real data appears once F02-F05 are merged.)

## Done when
- One POST /design runs analyze -> plan -> scout -> render and returns the full payload.
- A session is stored containing room_b64, space_brief, prompt, budget, products.
- It works against stubs AND against the real agents once merged.
- Merged into dev via PR "F06 orchestrator".
